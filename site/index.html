The morning shift should not start from zero.

ShiftMemory gives a team one shared memory for what changed, what still needs attention, and why it matters. A worker leaves a note. The next shift gets a cited handoff. A supervisor fixes mistakes. Cognee keeps that context usable across sessions and agents.

Why this exists

Handoffs fail when the next person has to reconstruct the past.

Most tools only work after someone gathers all the context. Real shift work is the opposite. The context is already scattered across notes, files, small decisions, and things people forgot to repeat.

ShiftMemory is not trying to be a medical assistant. It is a workflow memory system: keep the useful facts, cite where they came from, let a human correct them, and remove what should not stay.

The product flow

A simple workflow for the user. A real memory lifecycle underneath.

  1. 1

    Leave the shift note

    The worker writes what changed. To them, it feels like normal note-taking.

    Cognee remembers it as case memory.
  2. 2

    Generate the next handoff

    The next shift asks for the handoff and sees a short, source-backed brief.

    Cognee recalls the right context.
  3. 3

    Correct what matters

    A supervisor marks a detail important or fixes the way it was summarized.

    Cognee improves future recall.
  4. 4

    Remove what should not stay

    Incorrect, stale, or sensitive details can be removed from future handoffs.

    Cognee forgets that memory.
Where Cognee fits

Cognee is the part that makes the product more than a prompt.

The UI stays human: notes, handoffs, questions, corrections, deletion. The backend maps those actions to Cognee's memory operations and keeps the LLM on a short leash: only authorized, recalled case context goes into a generated answer.

User adds a note or file remember
User asks "what changed?" recall
Supervisor marks what mattered improve
Team removes bad or stale context forget
For reviewers

The submission separates product experience from engineering proof.

Normal users should not have to know what Cognee is. They should know whether the handoff is useful and where each claim came from.

Judges and engineers get the proof: case-scoped datasets, backend-only API keys, source references, memory operation traces, deletion behavior, and SRE runbooks.
