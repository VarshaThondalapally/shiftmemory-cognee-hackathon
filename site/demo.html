Run the real product, not the old mock.

The live product is the React and FastAPI app in the repository. This page is the judge runbook: configure Cognee and Gemini, start the backend and frontend, then open the Proof tab to inspect every memory call.

Final stack Gemini + Cognee + FastAPI

Reasoning: Gemini understands notes and creates recall plans.

Memory: Cognee remembers, recalls, improves, and forgets notes.

Verifier: backend accepts only source-backed handoff items.

Setup

Configure final mode in apps/backend/.env.

  1. 1

    Add secrets locally

    Use Cognee and Gemini keys only on the backend. Do not put secrets in the browser or GitHub Pages.

    MEMORY_BACKEND=cognee
  2. 2

    Start backend

    cd apps/backend then run uvicorn with the project virtualenv.

    http://127.0.0.1:8001
  3. 3

    Start frontend

    cd apps/frontend then run the Vite dev server.

    http://127.0.0.1:5173
  4. 4

    Open Proof

    Add a note, generate a handoff, prioritize a source, remove a source, then inspect the communication trace.

    redacted Cognee calls
Backend environment

Final mode values

The submitted code still supports local fallback for tests, but the hackathon proof run should use the live configuration below. 

MEMORY_BACKEND=cognee
COGNEE_BASE_URL=https://tenant-your-tenant-id.aws.cognee.ai
COGNEE_API_KEY=your_key
COGNEE_TENANT_ID=your_tenant_id
COGNEE_STRICT=true
COGNEE_DATASET_PREFIX=handoff-demo
COGNEE_ALLOW_LOCAL_RANK_FALLBACK=false
LLM_PROVIDER=gemini
GEMINI_API_KEY=your_key
