# Demo Video Script

Target length: 1:50.

This is the script used for the replacement video. The visuals are actual screenshots captured from the running product, not mockups.

## 0:00-0:10 Opening

Voiceover:

"This is Caregiver Handoff Memory. It is built for home-care and assisted-living teams where the next shift can miss small details that matter."

On screen:

- Public landing page.
- Product claim.

## 0:10-0:25 Product Roles

Voiceover:

"The product is not one shared note screen. Each person gets their own workspace. The supervisor assigns workers, the night caregiver records what changed, the morning lead receives the handoff, and judge mode shows the memory proof."

On screen:

- Login screen.
- Four demo roles.

## 0:25-0:40 Supervisor Assignment

Voiceover:

"First, the supervisor assigns Nia to Avery's night shift and Omar to the morning shift. Those assignments decide which care recipients each person can access after login."

On screen:

- Supervisor dashboard.
- Assignment rows.
- JWT/RBAC callout.

## 0:40-1:00 Remember

Voiceover:

"During the night, Nia saves one note: Avery woke twice after 3 AM, asked for water, and settled when the room stayed quiet. The backend sends this as durable memory to Cognee."

On screen:

- Night caregiver note capture.
- Remember action.
- Source ID callout.

## 1:00-1:20 Recall

Voiceover:

"In the morning, Omar does not paste that note. He clicks generate handoff. The backend asks Cognee for relevant case memory, Gemini turns the recalled sources into a handoff, and the app cites where each item came from."

On screen:

- Morning lead handoff.
- Before 9 AM, Watch Today, Preferences, Later Today.
- Sources used.

## 1:20-1:32 Ask Follow-Up

Voiceover:

"Omar can also ask a follow-up, like what changed after 3 AM. The answer is built from saved notes, not from a fresh explanation typed into the prompt."

On screen:

- Follow-up question.
- Source-backed answer.

## 1:32-1:42 Improve And Forget

Voiceover:

"The supervisor can mark a note important so future handoffs prioritize it, or remove stale memory so it stops appearing. That is the full memory lifecycle: remember, recall, improve, and forget."

On screen:

- Supervisor review controls.
- Keep important.
- Remove stale.

## 1:42-1:50 Proof Mode

Voiceover:

"For judges, proof mode shows the backend-to-memory trace: Cognee Cloud is active, Gemini is active, keys are redacted, and each memory operation is visible with sources."

On screen:

- Proof mode.
- Cognee Cloud and Gemini badges.
- Redacted communication trace.

The close is included in proof mode: Codex is disclosed for development, Gemini is the reasoning layer, and Cognee is the memory layer.
